The app is now transitioning toward a full product/series/dependency policy model. Use the steps below when adding new schedules like Pneumo.
1. Create concrete administered items in Products, including whether each one is currently available.
2. Create the clinical protection track in Series.
3. Link all allowed products to the series and add slot rules for each product path.
4. Add Dependencies for cross-series timing, such as "Pneumo after DTP + 15 days".
5. Use the legacy tabs only when you still need to maintain old schedule/group policy during the migration.
