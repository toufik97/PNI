1
Products

A Product is the concrete thing you stick in the arm (e.g., "Penta" or "DTC"). Products define availability and live/dead status.

  • Link to Legacy Vaccines
  • Track current stock/availability
  • Define live status for spacing
2
Series

A Series is a clinical goal (e.g., "DTP Protection"). It groups multiple products and defines the track rules.

  • Manage slot numbers (1, 2, 3...)
  • Define mixing policies (Strict/Flexible)
  • Version your clinical logic
3
Rules

Rules define the "When" and "Where". Each slot has specific age floors, targets, and intervals.

  • Age floors (Min Age)
  • Recommended dates
  • Overdue (Missing) thresholds

How It Works: The Decision Engine

The PNI engine doesn't just look for "the next dose". It evaluates all active series for a child and determines which slots are eligible, due, upcoming, or blocked.
Slot Resolution Path
  1. Filter by Age: For each series, identify the next unfinished slot (e.g., Slot 2). Look for rules matching the child's current age.
  2. Availability Check: If multiple products are valid for the current age (e.g., Penta and DTC), the engine prioritizes Available products.
  3. Continuity Check: If the child started with Product X, Transition Rules determine if switching to Product Y is allowed (due to age or outages).
  4. Dependency Evaluation: Even if a dose is due by age, it might be Blocked until another series reaches a specific slot (e.g., "Wait for DTP1").
Status Definitions
Due Today Recommended age met and all constraints clear.
Upcoming Not yet age-eligible, or waiting for an interval.
Missing Past the "overdue" age threshold without a record.
Blocked Clinically ready, but waiting for a Dependency or Stock.
Invalid Given too early or at a wrong interval.

Tutorial: Adding a New Vaccine (e.g. Pneumo)

Phase 1: Defining Products

Go to the Products tab. Create products for each brand (e.g., "PCV13", "PPSV23"). Mark if they are currently in stock. Stock status can be changed anytime without editing the clinical policy.

Phase 2: Creating the Track

Go to Series and create "Pneumo Series". Set the Mixing Policy to "Flexible" if brands can be interchanged, or "Strict" if you must finish with what you started.

Phase 3: Slot Rules

Inside the Series, add rules for Slot 1, 2, and 3. Each rule links a product to an age window (e.g., Slot 1 at 2 months). Tip: If Slot 1 can be either PCV13 or PCV15, add TWO rules for slot 1.

Phase 4: Safety Spacing

If Pneumo must wait for a DTP dose, go to Dependencies. Add a rule: Dependent Series: Pneumo / Anchor: DTP / Offset: 14 days. This ensures the scheduler never double-books them too close.
